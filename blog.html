JavaScript has a runtime model based on an event loop, which is responsible for executing the code, collecting and processing events, and executing queued sub-tasks.

The Event Loop occurs on JavaScriptâ€™s asynchronous programming which makes possible for JS to operate in multi-threads by following some steps:

  • Heap: Objects are allocated in a 'heap' which is just a name to donate a large region of memory.
  • Stack:Keep track of all the operations in line to executed. Whenever a function is ended, it pops from the 'stack'. It follows Last In First Out method.
  • Browser API: Hold or Store Asynchronous functions for an interval or a specific delay time which is set by setTimeout or setInterval.
  • Call Back Queue: When the interval is finished the function comes from 'browser API' to 'queue' where data will be waited until the 'stack' is empty means there is no synchronous data left.
  • Event Loop: help the function or data to pass to the 'stack' from 'queue' when it is fully empty. All Asynchronous data passes to the 'stack' through looping.

Undefined means a variable has been declared but has not yet been assigned value.

Way to get Undefined:

  1. Variable that is not initialized.
  2. Function with no return will be undefined.
  3. Function parameter that is not passed will be undefined.
  4. In a function, if return has nothing on the right side will return undefined.
  5. Property that doesn't exist on a object.
  6. Accessing array elements outside of the index range.
  7. Deleting an element inside an array using delete.
  8. set a value directly to Undefined.
